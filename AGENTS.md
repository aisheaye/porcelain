# AGENTS.md

## Project Overview

This repository is a porcelain auction data search project focused on crawling:

1. auction index/search result pages
2. lot detail pages
3. related images

The current codebase is a local Python-first scraper centered on Artron auction data. Data is stored in SQLite and downloaded images are cached on disk.

## Current Repository Shape

- `artron_scraper5.py`: main crawler for index/detail/image ingestion
- `detail_coverage_report.py`: reporting script for detail-field coverage
- `porcelain_auction.db`: local SQLite database generated by crawler
- `images/`: downloaded lot images
- `cookie.txt`: local cookie material required for crawling

## Working Rules

- Treat this repo as a data collection project, not a generic web app.
- Prefer small, direct Python changes over adding framework complexity.
- Preserve crawler idempotency. Re-runs should update status fields instead of duplicating records.
- Do not commit secrets, cookies, local databases, or downloaded images.
- Do not hardcode machine-specific absolute paths.
- Keep scripts runnable from the repository root.

## Data Safety

- `cookie.txt` is sensitive local state and must stay untracked.
- `porcelain_auction.db` is a generated artifact and must stay untracked.
- `images/` is cache/output data and must stay untracked.
- Schema changes in `auction_records`, `record_keywords`, or `crawl_queries` must remain backward-compatible with existing local data when feasible.

## Expected Commands

- Run crawler: `python artron_scraper5.py`
- Run coverage report: `python detail_coverage_report.py`

If new scripts are added, keep invocation similarly simple from repo root.

## Code Guidelines

- Prefer standard library plus minimal dependencies.
- Keep database schema definitions and migrations explicit in code.
- When adding extracted detail fields, update both ingestion and reporting paths.
- Log crawl failures with enough context to resume work without reprocessing everything.
- Favor resumable status fields like `pending`, `done`, and error markers over destructive resets.

## Git / GitHub Conventions

- Commit source code and lightweight docs only.
- Ignore generated crawl outputs and local credentials.
- Before pushing, verify no large binary artifacts or private cookie data are staged.

## When Extending The Project

- If adding new scrapers, isolate site-specific parsing logic clearly.
- If adding search capability, keep a strict boundary between raw crawl data and derived search indexes.
- If adding exports, place them in a dedicated generated-output path and ignore them unless the team explicitly wants versioned fixtures.
